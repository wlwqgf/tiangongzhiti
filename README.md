# 天工智梯 · 智能工厂申报全流程 AI 智能体系统

面向拟申报 **智能工厂梯度培育**（基础级 → 先进级 → 卓越级 → 领航级）的制造企业，提供从
**自测画像、政策问答、大纲生成、申报书研判、政策包测算** 到 **专家评定反馈** 的一站式 AI 辅助。
企业端 / 专家端 **双智能体物理隔离**（专家端含评审权重，企业端不可见，防泄题式刷分）。

> 定位：大连市智能制造产业协会「AI 专家顾问」——线上引流 + 线下专家交付。输出为**指导性意见**，非权威结论。

---

## 一、快速开始

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 配置模型接口（OpenAI 兼容，支持智谱/DeepSeek/通义等国产模型，可选）
cp .env.example .env
#   编辑 .env 填入 OPENAI_API_KEY / OPENAI_BASE_URL / MODEL_NAME

# 3. 启动（单命令，浏览器自动打开 http://localhost:8501）
bash run.sh
#   或： streamlit run app.py
```

**登录**：企业端账号 `01` / 密码 `11111`；专家端账号 `02` / 密码 `22222`（演示账号，可改 `app.py` 的 `CREDENTIALS`）。

**离线能力**：未配置模型接口时，「① 自评材料清单」「② 政策标准问答」「⑦ 政策包推荐」「⑤ 专家评定反馈」可完全离线运行
（本地清单 / 知识库 + 意图识别 / 本地政策求解器 / 关键词 Mock 引擎）；「③ 申报书大纲生成」「④ 申报书研判优化」
会在调用时给出配置提示，配置后即通过 OpenAI 兼容接口生成完整报告。

**数据流（串联各模块）**：① 画像推荐 → 输出当前/目标等级 → ② 政策问答随时答疑 →
③ 依等级生成大纲/缺口 → ④ 对草稿做六步研判 → ⑤ 专家基于研判做最终评定与反馈 →
⑦ 政策包按企业画像测算可叠加奖补，形成「企业自检 → AI 生成 → 专家把关 → 资金测算」的闭环。

## 三、功能模块总览

| 页面 | 功能 | 类型 |
|------|------|------|
| ① 自评材料清单 | 六维度评分、四级达标度、冲稳保概率、改造路径与成本；内置官方自评估要素条件与 c3mep 跳转 | 企业端·画像 |
| ② 政策标准问答 | 等级对标/指标/奖补/流程/政策/案例，离线意图识别 + 知识库问答 | 企业端·问答 |
| ③ 申报书大纲生成 | 10 模块问卷、缺口分析、分级大纲、F1-F8 | 企业端·生成 |
| ④ 申报书研判优化 | 六步预审、合规/一致性/证据评估、三级标记 | 企业端·研判 |
| ⑤ 专家评定反馈 | 初筛(PASS/REJECT/PENDING)→增强标注→6 维度 100 分评分 + P0/P1/P2 建议 | 专家端 |
| ⑥ 线下服务对接 | 协会线下专家交付引流单（企业信息留资） | 通用 |
| ⑦ 政策包推荐 | 最优不冲突政策组合求解 + 资金测算（对齐《惠企政策30条》真实奖补值） | 企业端·测算 |
| ⑧ 双端分离与合规 | 双智能体物理隔离说明 + 合规红线声明 | 通用·合规 |

> 双端分流：企业端可见 ①~④⑥⑦，专家端可见 ⑤；企业端问及评审权重/打分细则将被集中式护栏拦截并引导至专家端。

## 四、数据与政策依据

- 政策框架：《智能工厂梯度培育管理办法》（工信部联通装〔2025〕262号）、《要素条件（2025年版）》、《智能制造典型场景参考指引（2025年版）》（8 大环节 40 场景）、GB/T 39116-2020 成熟度模型。
- 知识库来源：IMA 共享知识库「智能工厂政策」及《惠企政策30条》（大连市）等公开政策，移植自各板块原始素材。
- 奖补标准（模块⑦，对齐惠企30条真实值示意）：国家级小巨人 50 万 / 省级专精特新 30 万 / 数字化车间 ≤30 万 / 智能工厂国家级 ≤200 万·省级 ≤100 万 / 工业设计中心国 50 省 30 / 单项冠军 ≤100 万 / 新型技改设备 10%·新型 20% 封顶 200 万 / 高企阶梯 5-20 万 + 所得税 15% / 国债 15% / 研发加计 100% / 先进制造增值税 5%。互斥组取高、叠加有上限，以当年政策为准。

1. **专家端（模块⑤）已用整合包真实内容对齐**：用户已直接提供「天工智梯_智能体整合包」（V3.3）全部文件，

   提示词（01/02/03）已逐字转录至 `core/e_prompts.py`，知识库与 40 场景、6 维度评分表、结论映射一并内置；
   离线关键词引擎（`pages/5_*.py` 内 `analyze_doc/screening/enhancement_skeleton/build_suggestions`）复现整合包 `05_参考实现`
   的 Mock 逻辑（含"替代人工操作"不判手工生产等负向词排除），**无需 API Key 即可开箱运行**；配置接口后三大模块可调用真实大模型生成完整报告。
   整合包真实素材置于 `expert_pack/` 目录供透明核查。
2. **政策阈值与案例** 来自各板块已成功下载的本地素材；网盘侧对部分文件当前返回密钥失效，

3. **逐级申报**：四级梯度原则上不可跳级，跳级面临一票否决；系统在各模块均强制提示。
4. **免责声明**：所有输出均标注「AI 生成、仅供参考，最终以专家评定及工信部门审核为准」。

### 方式二：下载源码压缩包

不会用 Git 也没关系：在 GitHub 仓库页点 **Code → Download ZIP**，即可拿到**不含虚拟环境** 的纯源码包（约几百 KB），解压后 `pip install -r requirements.txt && bash run.sh` 即可运行。

### 提交改动流程

1. 在 GitHub 上 **Fork** 本仓库，或联系维护者加入协作者；
2. 本地新建分支：`git checkout -b feature/你的改动`；
3. 改完后先验证语法：`python -m py_compile app.py core/*.py pages/*.py`；
4. 提交并推送到你的分支，发起 **Pull Request**。

### 必须遵守的工程约定（红线）

完整版见 **`HANDOFF.md`**（接手文档），核心要点：

- **零Key离线可跑**：未配置模型接口时各模块必须能跑（离线规则引擎兜底）；新增功能须同时提供"大模型版"和"离线版"两条路径。
- **选案例前全空 / 点生成才出报告 / 点击才显示内容**：报告只在点击按钮后出现，不能一进页面就显示。
- **防代码泄露**：保留 `.streamlit/config.toml` 的 `showErrorDetails=false`，页面禁止 `st.json()` 暴露内部字段名。
- **双端分流**（企业端 ①~④⑥⑦ / 专家端 ⑤）与**隐藏原生侧边栏导航**不能破坏。
- 标题不得用 `background-clip:text` 透明填充（会让 emoji 图标消失）。

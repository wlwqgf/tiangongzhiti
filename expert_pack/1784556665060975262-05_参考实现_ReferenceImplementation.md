# 05 · 参考实现（Reference Implementation）

> 用途：无 LLM API Key 时的关键词 Mock 引擎逻辑。可用于部署演示版、离线测试、或对接收方的「能力验证」。
> 对应源码：`smart-factory-agent/web/js/mock.js`（纯前端） 与 `smart-factory-agent/app/services/mock_service.py`（后端）。

---

## 一、设计目标
在不调用大模型的情况下，用**确定性关键词分析**复现三大模块的判定逻辑，使整合包「开箱即用」、便于对方验证接口与输出格式。生产环境应替换为本包 `01/02/03` 驱动的真实 LLM。

## 二、核心分析函数 `_analyzeDoc(text, targetLevel)`
从申报书全文提取以下布尔/语义信号（正则 + 关键词匹配，注意负向词排除，如"替代人工操作"不算手工生产）：

| 信号 | 命中关键词 | 用途 |
|------|-----------|------|
| `is_small_scale` | 小微企业/小规模/年营收低于XX万 | 基础要求①反向 |
| `is_manual_production` | 手工生产 /（非"替代""取代"的）人工操作 / 传统制造 / 人工装配为主 | 基础级否决 |
| `has_mes` | MES / 制造执行系统 | 先进级强制 |
| `has_iot` | 物联网 / 工业网关 / 设备联网 / 数据采集 | 数据互通 |
| `has_automation` | 自动化 / 数字化改造 / 关键工序自动化 | 基础级 |
| `has_digital_design` | CAD / CAM / 数字化设计 / 仿真 | 先进级 |
| `has_ai` | AI / 人工智能 / 机器视觉 / 数字孪生 / 智能算法 | 卓越/领航 |
| `has_datashare` | 数据互通 / 集成贯通 / 系统对接 / ERP对接 | 先进级 |
| `has_safety_env` | 安全能源环保 / 能耗监测 / 环保管控 | 先进级 |
| `covered_environments` | 统计命中 工厂建设/研发设计/生产作业/生产管理/运营管理 的环节数 | 覆盖环节 |
| `has_revenue` / `has_employees` / `has_commitment` | 营收/员工/承诺 关键词 | 形式审查 |
| `maturity_score` | 从"成熟度自评估 X.X"或"三级/二级"提取数值 | 门槛判定 |

## 三、三大模块映射
- **screening**：形式审查(4类材料) → 基础要求5条 → 目标等级一票否决(见02模块一) → 输出 `PASS/REJECT/PENDING_EXPERT` + 理由
- **enhancement**：按五大环节生成对照表骨架 + 40场景覆盖矩阵（基于 `covered_environments` 与关键词命中推断匹配状态）+ 红/黄/蓝问题分级 + 成熟度对标
- **scoring**：接收专家6维度分数 → 按结论映射(≥80/65-79/50-64/35-49/<35)生成《修改建议报告》+ P0/P1/P2 清单（基于 enhancement 的问题分级填入动作级模板）

## 四、验证结果（3样本）
| 样本 | 目标等级 | Mock 判定 | 预期 |
|------|---------|----------|------|
| 华锐重工 | 先进级 | PASS | ✅ |
| 新兴机械 | 先进级 | REJECT（建议降级） | ✅ |
| 精密电子 | 基础级 | PASS | ✅ |

> 注意：样本3 原文"实现全自动化，替代人工操作"曾被误判为手工生产，已通过负向词排除修复（前端 mock.js 与后端 mock_service.py 同步）。

## 五、接入建议
- 对方若用本包做**能力验证**：直接跑 mock 引擎，确认 I/O schema 与 `智能体卡片_agent.card.json` 一致即可。
- 对方若做**生产集成**：把 `_analyzeDoc` 的关键词信号作为 LLM 的「检索提示」拼进 prompt，可提升真实模型判定准确率与可解释性。

# 天工智梯 · 智能体整合包（Integration Pack）V3.3

> 本包用于把「天工智梯（智能工厂梯度培育申报专家顾问智能体）」作为**一个能力子模块**，
> 整合进更大的智能体系统 / 多智能体编排 / 工作流平台。
> 接收方只需读取本包，即可重新实例化出该智能体的全部行为，无需原始项目代码。

---

## 一、这个智能体是干什么的

- **定位**：服务于智能制造梯度培育申报场景的「专家端 AI 顾问」
- **使命**：辅助评审专家完成企业《智能工厂申报书》的评审，覆盖三大任务模块
- **输出性质**：仅供专家参考，不是最终评审结论（所有标准引用须标注出处文件名）
- **政策依据**：工信部联通装〔2025〕262号 + 《要素条件（2025年版）》+ GB/T 39116/39117 + 6维度100分评分表

### 三大能力模块（整合时的「工具 / 子智能体」边界）
| 模块 | 触发时机 | 输入 | 输出 |
|------|---------|------|------|
| ① 初步筛查 | 企业上传申报书草稿后、送专家前 | 申报书全文 + 目标等级 | `PASS / REJECT / PENDING_EXPERT` 结构化判定 + 拦截理由 |
| ② 内容增强标注 | 初筛通过后、专家阅读前 | 申报书全文 + 目标等级 | 《申报书增强分析报告》（原文-标准-关键词三方对照 + 场景覆盖矩阵 + 问题分级） |
| ③ 评分辅助与建议 | 专家评完分后 / 申报书不合格时 | 专家评分 + 增强报告 + 原文 | 《申报书修改建议报告》（6维度评分明细 + P0/P1/P2 修改清单 + 能力建设建议） |

## 三、整合到别的智能体时，有 3 种常见接法

### 接法 A：作为「子智能体 / 专家角色」被编排调用
适用于多智能体框架（如 AutoGen、LangGraph、CrewAI、Dify、Coze 等）。
- 把 `01_系统提示词` 作为该子智能体的 system prompt
- 把 `02_能力契约` 的 3 个模块拆成 3 个 tool / 3 个 sub-agent
- I/O schema 见 `智能体卡片_agent.card.json → capabilities[]`
- 主智能体按流程路由：筛查 →（通过）→ 增强 →（专家评分）→ 建议生成

### 接法 B：作为「单一 system prompt」塞进更大的通用智能体
适用于只用一个大模型、想让它身兼多职的场景。
- 把 `01_系统提示词` + `03_知识库` 拼进主智能体的 system prompt
- 在 `03_知识库` 末尾追加「当你收到申报书 + 目标等级时，执行①筛查→②增强→③评分三步走」的路由指令
- 缺点：上下文变长，复杂任务易跑偏；适合轻量整合

### 接法 C：作为「独立 API 服务」被调用
适用于已有后端、想远程调用的场景。
- 原始项目（`smart-factory-agent/`）已是 FastAPI 服务，暴露 `/api/screening`、`/api/enhancement`、`/api/scoring`
- 把该服务部署后，别的智能体通过 HTTP 工具调用即可
- 需要配置真实 LLM API Key（默认走 Mock 降级）

## 五、整合时的 4 个注意点

1. **知识版本**：本包知识锚定《要素条件（2025年版）》+ 辽宁省2026申报细则，**卓越级/领航级评分表、其他省份细则尚未补全**（见 V3.3 待办）。整合前请确认对接系统所需的省份/等级是否已覆盖。
2. **接口幂等**：三个模块建议设计为独立可重入（同一申报书可重复跑筛查/增强）。
3. **人机协同边界**：该智能体只出「建议」，最终结论必须由专家确认——整合时务必保留「专家审核」节点，不要让它自动出终审结论。
4. **标准出处**：输出必须带 `标准出处`（如《要素条件》先进级生产作业），对接系统若要做可溯源展示，可要求严格 JSON 输出（见 `02_能力契约` 模块一的 JSON schema）。
